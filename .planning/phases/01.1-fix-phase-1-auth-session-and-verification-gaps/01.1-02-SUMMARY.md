# Phase 01.1 Plan 2 Summary

**Strict JWT configuration and parsing, locked single-use refresh rotation, and controller security tests running through the production bearer filter chain**

## Performance

- **Duration:** 18 min

## Accomplishments
- Removed the insecure JWT secret fallback and moved issuer/secret validation into `SecurityProperties`, making startup fail closed when auth config is unsafe.
- Changed refresh rotation to lock the existing refresh-session row so concurrent redemption of one token produces exactly one winner and one reuse failure.
- Replaced the `permitAll()` controller-test bypass with real bearer-filter assertions and a 401 authentication entrypoint for protected routes.

3. **Task 3: Replace security-test bypasses with real filter-chain assertions** - `e737152` (fix)

## Files Created/Modified
- `src/main/java/com/collabeditor/auth/security/SecurityProperties.java` - Validates JWT secret and issuer as explicit application configuration.
- `src/main/java/com/collabeditor/auth/service/JwtTokenService.java` - Issues tokens from validated config and parses them into `Optional<Claims>` with issuer and subject checks.
- `src/main/java/com/collabeditor/auth/security/JwtAuthenticationFilter.java` - Treats invalid bearer claims as authentication failure instead of throwing on malformed UUID subjects.
- `src/main/java/com/collabeditor/auth/persistence/RefreshSessionRepository.java` - Adds a pessimistic-lock refresh lookup for atomic rotation.
- `src/main/java/com/collabeditor/auth/service/RefreshSessionService.java` - Uses the locked lookup and records last use when a refresh chain is consumed.
- `src/main/java/com/collabeditor/auth/security/SecurityConfig.java` - Returns 401 for unauthorized protected routes through an explicit entrypoint.
- `src/main/resources/application.yml` - Declares JWT secret and issuer as first-class app security properties.
- `src/test/java/com/collabeditor/auth/JwtTokenServiceTest.java` - Covers wrong issuer, malformed subject, expired token, and blank/short secret configuration.
- `src/test/java/com/collabeditor/auth/RefreshSessionServiceTest.java` - Adds a concurrent single-use refresh race test.
- `src/test/java/com/collabeditor/auth/AuthControllerTest.java` - Exercises protected-route behavior through the real security filter chain.
- `src/test/java/com/collabeditor/session/SessionControllerTest.java` - Moves the session controller slice onto the production security config after the test bypass was removed.

## Decisions Made
- Used strict configuration validation at bean construction time instead of deferred runtime checks so unsafe JWT configuration cannot boot.
- Locked the refresh-session row instead of trying to infer single-use semantics from best-effort read-check-write logic.
- Verified bearer-token behavior against the real `/api/sessions` endpoint so the protected-route tests reflect actual application wiring.
